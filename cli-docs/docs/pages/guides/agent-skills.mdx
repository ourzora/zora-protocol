# Agent Skills

Pre-built trading strategies that compose Zora CLI commands into automated workflows. Each skill is a markdown prompt file an agent follows step-by-step — ship it to Claude Code, Cursor, Windsurf, or any LLM that can read markdown and invoke shell commands.

## Available Skills

| Skill          | Category  | Description                                                              |
| -------------- | --------- | ------------------------------------------------------------------------ |
| `/copy-trader` | Social    | Mirror another user's trades — existing holdings, future trades, or both |
| `/early-buyer` | Discovery | Auto-buy new coin launches from creators you follow                      |
| `/watchlist`   | Discovery | Track coins and alert when market cap hits configured thresholds         |
| `/take-profit` | Risk      | Auto-sell positions at configured take-profit or stop-loss price targets |

Every skill runs as a single iteration per invocation and persists state to a local JSON file. Re-invoke the skill to run the next cycle, or use the agent's native scheduling (see [Scheduling](#scheduling) below).

## Prerequisites

Skills invoke the Zora CLI. No install required — use it via `npx`:

```bash
npx @zoralabs/cli --version
```

Run `npx @zoralabs/cli setup` once to configure a wallet. To avoid the `npx` prefix everywhere, install globally: `npm install -g @zoralabs/cli`.

## Installation

Every skill is a markdown prompt file. Install a skill by giving its URL to the agent — the agent fetches the file at runtime and follows its instructions.

Skill URLs:

- `/copy-trader` — https://raw.githubusercontent.com/ourzora/zora-protocol/main/packages/cli/skills/copy-trader/SKILL.md
- `/early-buyer` — https://raw.githubusercontent.com/ourzora/zora-protocol/main/packages/cli/skills/early-buyer/SKILL.md
- `/watchlist` — https://raw.githubusercontent.com/ourzora/zora-protocol/main/packages/cli/skills/watchlist/SKILL.md
- `/take-profit` — https://raw.githubusercontent.com/ourzora/zora-protocol/main/packages/cli/skills/take-profit/SKILL.md

### Claude Code / Cursor / Windsurf

Add the skill URL to your project's agent instructions (`CLAUDE.md`, `.cursorrules`, `.windsurfrules`, etc.):

```markdown
Fetch and follow the copy-trader skill:
https://raw.githubusercontent.com/ourzora/zora-protocol/main/packages/cli/skills/copy-trader/SKILL.md
```

The agent fetches the file on first use and runs the skill. Repeat per skill.

### Custom Agents

Same pattern — include the URL (or the fetched markdown) in the agent's system prompt. The agent needs the ability to fetch URLs and invoke shell commands.

## Scheduling

Skills run once per invocation. To poll on an interval, use the scheduler for the agent:

- **Claude Code** — `/loop 30m /copy-trader` invokes the skill every 30 minutes
- **Cursor / Windsurf** — use their scheduled task / cron-equivalent to re-invoke the skill command
- **Cron / shell** — script a shell loop or cron entry that runs `zora` commands directly using the same prompt file as a reference
- **Manual** — just re-invoke the skill when you want the next cycle

State persists in files like `.copy-trader-state.json` in the working directory, so each invocation picks up where the last left off. Add these patterns to `.gitignore` to avoid committing them.
